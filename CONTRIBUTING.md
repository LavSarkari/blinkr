# 🤝 Contributing to Blinkr

First off, thank you for considering contributing to Blinkr! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## 🚀 How Can I Contribute?

### Reporting Bugs
- Use the **GitHub Issues** tab to report bugs.
- Provide a clear, descriptive title.
- List the steps to reproduce the issue.
- Describe the behavior you observed and what you expected to happen.

### Suggesting Enhancements
- If you have an idea for a new feature or a UI/UX improvement, open a **Feature Request** issue!
- Explain why the enhancement would be useful and how it aligns with the Blinkr vision.

### Pull Requests
1. **Fork the repo** and create your branch from `main`.
2. Ensure your code follows the existing style and is well-documented.
3. If you've added code that should be tested, add tests!
4. Ensure the build passes (`npm run build`).
5. Open a Pull Request with a clear title and description of your changes.

---

## 🎨 Creative Guidelines

- **Keep it Premium**: Blinkr is known for its high-end AMOLED aesthetic. Use fluid animations and refined color palettes.
- **Privacy First**: Any new feature must respect absolute user anonymity and zero data retention.
- **Real-time Efficiency**: Optimize all broadcasts and presence updates to keep the "Scanning Network" fast.

---

## 🛠️ Development Setup

```bash
# Install dependencies
npm install

# Start dev server
npm run dev

# Build for production
npm run build
```
